# ⚡ GITHUB PAGES — QUICK SETUP (5 Minutes)

## 🎯 THE EASY WAY (No Command Line)

### **Step 1: Create GitHub Repository** (1 min)
1. Go to https://github.com/new
2. Name: `sice-ops`
3. Make it **Public** (required for free Pages)
4. Click **Create repository**

### **Step 2: Rename Your File** (30 sec)
- Rename: `SICE_OPS_V3_FIXED.html` → `index.html`

### **Step 3: Upload File** (1 min)
1. Go to your new repo
2. Click **Add file** → **Upload files**
3. Drag `index.html` into the upload area
4. Click **Commit changes**

### **Step 4: Enable GitHub Pages** (1 min)
1. Click **Settings** (top right)
2. Click **Pages** (left sidebar)
3. Under "Build and deployment":
   - Source: **Deploy from a branch**
   - Branch: **main**
   - Folder: **/ (root)**
4. Click **Save**

### **Step 5: Wait & Access** (2-3 min)
GitHub will show: "Your site is live at: https://USERNAME.github.io/sice-ops"

## 🎓 COMMON QUESTIONS

**Q: Why rename to index.html?**
A: GitHub Pages serves `index.html` as the home page automatically. Otherwise you'd need the long filename in the URL.

**Q: Is my code visible?**
A: Yes, your HTML is public. That's fine—it's front-end code. Don't put API keys or secrets in it.

**Q: Can I use a custom domain?**
A: Yes! Buy a domain (~$10/year) and connect it in Settings → Pages.

**Q: How often can I update?**
A: As many times as you want. Just push changes and GitHub Pages updates in 2-3 minutes.

**Q: What if I need a backend?**
A: GitHub Pages only hosts static files. For database/server features, use Firebase, Supabase, or similar services (some are free).

**Q: Will my app work offline?**
A: Yes! After first visit, it works offline (with Service Worker).
